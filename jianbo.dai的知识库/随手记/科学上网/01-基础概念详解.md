## 什么是科学上网

### 定义

**科学上网** 是指通过技术手段突破网络限制，访问被屏蔽或限制访问的互联网内容的方法。

```
┌─────────────────────────────────────────────────────────────┐
│                      正常访问流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   你的电脑 → GFW (防火墙) → 被阻挡 ❌                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                      科学上网流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   你的电脑 → 加密隧道 → 海外服务器 → 目标网站 ✅              │
│                  ↓                                         │
│              GFW 无法识别                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 核心原理

科学上网的本质是：**建立一条加密的隧道，让流量伪装或绕过检测**

```
明文流量 (被识别) ──加密──> 加密流量 (无法识别) ──解密──> 明文流量
    ↓                        ↓                      ↓
  本地                    网络传输                海外服务器
```

## 网络基础知识

### IP 地址

**IP (Internet Protocol)** 地址是设备在网络中的"门牌号"。

| 类型 | 说明 | 示例 |
|------|------|------|
| **IPv4** | 32位地址，约43亿个 | `192.168.1.1` |
| **IPv6** | 128位地址，几乎无穷 | `2001:db8::1` |

```bash
# 查看本机IP
Windows: ipconfig
Linux/Mac: ifconfig 或 ip addr

# 查看公网IP
curl ip.sb
curl ifconfig.me
```

### 端口 (Port)

端口是设备上应用程序的"通道编号"，范围 0-65535。

| 端口范围 | 用途 |
|----------|------|
| 0-1023 | 系统保留（如 80=HTTP, 443=HTTPS） |
| 1024-49151 | 注册端口 |
| 49152-65535 | 动态/私有端口 |

```
IP:端口 = 地址:门牌号
例如：192.168.1.1:443
     ↑          ↑
   服务器地址   HTTPS端口
```

### DNS (域名系统)

**DNS** 是互联网的"电话簿"，将域名转换为IP地址。

```
访问 www.google.com
        ↓
   DNS查询
        ↓
   返回 IP: 142.250.xxx.xxx
        ↓
    连接到服务器
```

| DNS类型 | 说明 | 示例 |
|---------|------|------|
| **普通DNS** | 明文查询，可被监控 | 8.8.8.8 (Google) |
| **DoT** | DNS over TLS，加密 | 1.1.1.1 (Cloudflare) |
| **DoH** | DNS over HTTPS，更隐蔽 | https://dns.google/dns-query |

### UDP vs TCP

| 特性 | TCP | UDP |
|------|-----|-----|
| **连接** | 面向连接（三次握手） | 无连接 |
| **可靠性** | 可靠传输，有重传 | 不可靠，可能丢包 |
| **速度** | 较慢 | 更快 |
| **用途** | 网页、文件传输 | 视频、游戏、直播 |

```
TCP: 你好→收到→开始传输（可靠但慢）
UDP: 直接发送（快但可能丢包）
```

## 代理与VPN的区别

### 什么是代理

**代理 (Proxy)** 是一个"中间人"，代替你去访问网站。

```
正常访问：
你 → 直接访问网站

代理访问：
你 → 代理服务器 → 网站
        ↑
    网站看到的是代理的IP
```

### 什么是 VPN

**VPN (Virtual Private Network)** 虚拟专用网络，在公共网络上建立加密通道。

```
VPN 隧道：
你 ──加密──> VPN服务器 ──解密──> 目标网站
    ↑                      ↑
  局域网内               真实访问
  其他人无法窥探
```

### 对比表

| 特性 | 代理 | VPN |
|------|------|-----|
| **工作层级** | 应用层（浏览器） | 系统层（全部流量） |
| **加密** | 可能不加密 | 必须加密 |
| **覆盖范围** | 仅配置的软件 | 全局流量 |
| **速度** | 较快 | 稍慢（加密开销） |
| **隐蔽性** | 一般 | 较高 |

### SOCKS5 代理

SOCKS5 是一种更底层的代理协议，支持TCP和UDP。

```
应用层协议 (HTTP, FTP, SMTP...)
         ↓
    SOCKS5 代理 ← 透明转发任何流量
         ↓
      目标服务器
```

## 加密与混淆技术

### 对称加密与非对称加密

| 类型 | 说明 | 速度 | 用途 |
|------|------|------|------|
| **对称加密** | 加密解密用同一把钥匙 | 快 | 数据传输 |
| **非对称加密** | 公钥加密，私钥解密 | 慢 | 密钥交换 |

```
对称加密:
你(钥匙) ──加密──> 密文 ──解密(同一把钥匙)──> 原文

非对称加密:
你(公钥) ──加密──> 密文 ──解密(私钥)──> 原文
  (给别人)                        (只有你有)
```

### TLS/SSL

**TLS (Transport Layer Security)** 是目前最主流的加密协议。

```
TLS 握手过程:
1. 客户端 → 服务器: 我要连接，支持的加密方法
2. 服务器 → 客户端: 这是我的证书和公钥
3. 客户端: 验证证书，生成会话密钥，用公钥加密发送
4. 服务器: 用私钥解密得到会话密钥
5. 双方: 使用会话密钥加密通信
```

### REALITY 技术

**REALITY** 是 Xray 推出的革命性技术，无需证书即可实现 TLS 伪装。

```
传统 TLS:
需要域名 → 申请证书 → 配置服务 → 可被检测到是自建服务

REALITY:
无需域名 → 伪造 TLS 握手 → 伪装成目标网站(如Apple) → 无法区分
```

**REALITY 工作原理：**

```
┌─────────────────────────────────────────────────────────────┐
│              REALITY 连接建立过程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 客户端连接到伪装目标 (如 www.apple.com)                 │
│  2. 在 TLS握手过程中注入 REALITY 握手                        │
│  3. 服务器识别并响应，建立加密连接                           │
│  4. 外部看起来就像正常访问 Apple 网站                        │
│                                                             │
│  优势: 无证书、抗封锁、高隐蔽                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 常见术语解释

### 服务器相关

| 术语 | 说明 |
|------|------|
| **VPS** | Virtual Private Server，虚拟专用服务器 |
| **云服务器** | 弹性计算服务，如 AWS、阿里云 |
| **独立服务器** | 整台物理服务器 |
| **CN GIP** | 中国大陆原生IP，容易被识别 |
| **Native IP** | 本地原生IP，更稳定 |

### 协议相关

| 术语 | 全称 | 说明 |
|------|------|------|
| **SS** | Shadowsocks | 轻量级代理协议 |
| **SSR** | ShadowsocksR | SS的增强版（已过时）|
| **V2Ray** | Project V | 综合性代理平台 |
| **Xray** | Xray-core | V2Ray的升级版，性能更好 |
| **Trojan** | - | 伪装成HTTPS流量的协议 |
| **VLESS** | Versatile Lightweight Extensible Security Setup | 轻量级协议 |
| **VMess** | V2Ray的加密协议 | 需要时间同步 |

### 传输相关

| 术语 | 说明 |
|------|------|
| **WebSocket (WS)** | HTTP协议升级，长连接 |
| **HTTP/2** | 二进制分帧，多路复用 |
| **gRPC** | Google的RPC框架，基于HTTP/2 |
| **QUIC** | 基于UDP的新传输协议，Hysteria2使用 |
| **XTLS** | TLS的加速版本，降低加密开销 |

### 配置相关

| 术语 | 说明 |
|------|------|
| **UUID** | 通用唯一标识符，用作客户端ID |
| **SNI** | Server Name Indication，服务器名称指示 |
| **CDN** | Content Delivery Network，内容分发网络 |
| **回落** | FallBack，非代理流量导向的网站 |
| **分流** | Routing，按规则将流量导向不同出口 |

### 客户端相关

| 术语 | 说明 |
|------|------|
| **订阅** | 节点信息的URL链接，自动更新配置 |
| **节点** | 单个代理服务器配置 |
| **机场** | 提供代理服务的商业服务商 |
| **TUN模式** | 虚拟网卡，接管系统所有流量 |
| **系统代理** | 只代理支持代理设置的软件 |

## 学习建议

### 推荐学习顺序

```
1️⃣ 理解基础概念 (本文档)
   ↓
2️⃣ 了解主流方案对比
   ↓
3️⃣ 选择一种方案动手实践
   ↓
4️⃣ 学习客户端配置和分流规则
   ↓
5️⃣ 深入了解原理和高级配置
```

### 实践建议

| 阶段 | 建议方案 |
|------|----------|
| **入门** | VLESS + REALITY (无需域名) |
| **进阶** | 学习 Clash 分流规则 |
| **高级** | 多服务器负载均衡、故障转移 |
