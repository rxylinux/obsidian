## 协议详细对比

### VLESS + REALITY

#### 技术原理

```
┌─────────────────────────────────────────────────────────────┐
│           VLESS + REALITY 工作流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  客户端                          服务端                      │
│     │                              │                        │
│     │  1. 发起 TLS 握手              │                        │
│     │  (伪装目标: www.apple.com)     │                        │
│     │ ─────────────────────────────> │                        │
│     │                              │                        │
│     │  2. REALITY 握手 (嵌入TLS)     │                        │
│     │ <───────────────────────────── │                        │
│     │                              │                        │
│     │  3. 建立加密连接               │                        │
│     │ ═════════════════════════════ │                        │
│     │                              │                        │
│  外部看起来: 客户端正在访问 Apple 网站                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 无需域名和证书 | ⚠️ 配置相对复杂 |
| ✅ 抗封锁能力强 | ⚠️ 部分老旧客户端不支持 |
| ✅ 流量伪装度高 | ⚠️ 需要选择合适的伪装目标 |
| ✅ 速度稳定 | |
| ✅ 支持多路复用 | |

#### 关键参数

```yaml
# Clash Meta 配置示例
proxies:
  - name: VLESS-Reality
    type: vless
    server: 服务器IP
    port: 443
    uuid: 你的UUID
    network: tcp
    tls: true
    flow: xtls-rprx-vision  # Vision 流控
    servername: www.cloudflare.com  # 伪装目标
    reality-opts:
      public-key: 公钥
      short-id: ""
```

#### 常用伪装目标

| 目标网站 | 特点 | 推荐度 |
|----------|------|--------|
| `www.cloudflare.com` | CDN巨头，流量常见 | ⭐⭐⭐⭐⭐ |
| `www.apple.com` | Apple流量，难以封禁 | ⭐⭐⭐⭐⭐ |
| `www.microsoft.com` | 企业流量，稳定 | ⭐⭐⭐⭐ |
| `time.com` | 简单稳定 | ⭐⭐⭐⭐ |
| `www.yahoo.com` | 老牌网站 | ⭐⭐⭐ |

### VLESS + TLS

#### 技术原理

```
┌─────────────────────────────────────────────────────────────┐
│            VLESS + TLS 工作流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  客户端                          服务端                      │
│     │                              │                        │
│     │  1. 正常 TLS 握手              │                        │
│     │  (使用真实域名证书)            │                        │
│     │ ─────────────────────────────> │                        │
│     │                              │                        │
│     │  2. 证书验证通过               │                        │
│     │ <───────────────────────────── │                        │
│     │                              │                        │
│     │  3. VLESS 协议握手             │                        │
│     │ ═════════════════════════════ │                        │
│     │                              │                        │
│  外部看起来: 访问某个 HTTPS 网站                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 成熟稳定 | ❌ 需要购买域名 |
| ✅ 真实证书 | ❌ 需要申请证书 |
| ✅ 兼容性好 | ⚠️ 域名可能被封锁 |
| ✅ 速度快 | |
| ✅ 可搭配 CDN | |

#### 证书申请方式

```bash
# 使用 acme.sh 自动申请证书
curl https://get.acme.sh | sh
~/.acme.sh/acme.sh --set-default-ca --server letsencrypt

# 申请证书
~/.acme.sh/acme.sh --issue -d yourdomain.com --nginx

# 安装证书
~/.acme.sh/acme.sh --install-cert -d yourdomain.com \
  --key-file /path/to/key.pem \
  --fullchain-file /path/to/cert.pem
```

### Hysteria2

#### 技术原理

```
┌─────────────────────────────────────────────────────────────┐
│            Hysteria2 协议栈                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  应用层: Hysteria2 协议                                      │
│     ↓                                                       │
│  传输层: QUIC (基于 UDP)                                     │
│     ↓                                                       │
│  特性:                                                      │
│  • 0-RTT 握手 - 更快连接建立                                 │
│  • 多路复用 - 避免队头阻塞                                   │
│  • BBR 拥塞控制 - 弱网优化                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 速度极快 | ❌ 基于 UDP，可能被 QoS |
| ✅ 低延迟 | ⚠️ 敏感期抗封锁弱 |
| ✅ 弱网优化 | ⚠️ 流量特征可识别 |
| ✅ 可选域名/非域名 | ⚠️ 部分网络环境不友好 |
| ✅ 吞吐量可达 2Gbps | |

#### 适用场景

```
✅ 适合:
   • 常规网络环境
   • 追求极速传输
   • 大文件下载
   • 弱网环境 (高延迟、丢包)

❌ 不适合:
   • 敏感时期
   • 严格管控的网络
   • 对 UDP 限速的运营商
```

### Trojan

#### 技术原理

```
┌─────────────────────────────────────────────────────────────┐
│            Trojan 工作原理                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Trojan = 真正的 HTTPS 流量                                  │
│                                                             │
│  正常 HTTPS:                                                 │
│  GET / HTTP/1.1 → TLS 加密 → 服务器                         │
│                                                             │
│  Trojan:                                                     │
│  密码验证 → TLS 加密 → 服务器 → 转发                         │
│       (在 HTTP 头部中)                                       │
│                                                             │
│  外部看起来完全一样，无法区分！                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 流量特征真实 | ❌ 需要域名和证书 |
| ✅ 隐蔽性高 | ⚠️ 配置稍复杂 |
| ✅ 性能好 | |
| ✅ 成熟稳定 | |

```
┌─────────────────────────────────────────────────────────────┐
│            Shadowsocks 工作流程                              │
├─────────────────────────────────────────────────────────────┤
```

| Shadowsocks 2022 | ⚠️ 可用 | 改进版，但仍不如新协议 |

### VMess

#### 技术原理

VMess 是 V2Ray 的原生协议，使用动态时间戳和 ID 进行认证。

```
┌─────────────────────────────────────────────────────────────┐
│            VMess 协议结构                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  数据包格式:                                                 │
│  ┌────────────────────────────────────┐                    │
│  │ 认证信息 | 时间戳 | 数据 | 检验和   │                    │
│  └────────────────────────────────────┘                    │
│                                                             │
│  特点: 动态端口、时间验证、加密强                             │
│  缺点: 特征明显，易被识别                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 现状

| 组合方式 | 状态 | 说明 |
|----------|------|------|
| VMess + TCP | ❌ 不推荐 | 特征极明显 |
| VMess + WebSocket | ⚠️ 可用 | 搭配 CDN 可用 |
| VMess + gRPC | ⚠️ 可用 | 相对隐蔽 |

## 传输方式对比

### WebSocket (WS)

```
特点:
• HTTP 协议升级
• 长连接，低开销
• 可与 Nginx 等反向代理结合
• 流量看起来像正常的 WebSocket

适用: VLESS+TLS, VMess
```

### HTTP/2

```
特点:
• 二进制协议
• 多路复用
• 头部压缩
• 流量看起来像正常的 HTTP/2

适用: VMess, Trojan
```

### gRPC

```
特点:
• 基于 HTTP/2
• Protocol Buffers 序列化
• 流量看起来像 gRPC 调用
• 相对隐蔽

适用: VMess, VLESS
```

### QUIC

```
特点:
• 基于 UDP
• 0-RTT 握手
• 多路复用
• 连接迁移

适用: Hysteria2
缺点: 运营商可能限制 UDP
```

## 安全与混淆

### TLS 版本对比

| 版本 | 状态 | 安全性 |
|------|------|--------|
| TLS 1.0 | ❌ 已弃用 | 低 |
| TLS 1.1 | ❌ 已弃用 | 低 |
| TLS 1.2 | ⚠️ 可用 | 中 |
| TLS 1.3 | ✅ 推荐 | 高 |

### 加密算法

| 算法 | 特点 | 推荐度 |
|------|------|--------|
| AES-128-GCM | 快，安全 | ⭐⭐⭐⭐ |
| AES-256-GCM | 更安全，稍慢 | ⭐⭐⭐⭐⭐ |
| ChaCha20-Poly1305 | 移动端优化 | ⭐⭐⭐⭐ |

### 混淆技术

| 技术 | 原理 | 效果 |
|------|------|------|
| **REALITY** | 伪造 TLS 握手 | ⭐⭐⭐⭐⭐ |
| **TLS 伪装** | 使用真实证书 | ⭐⭐⭐⭐ |
| **WebSocket** | 伪装成 WS 流量 | ⭐⭐⭐ |
| **HTTP/2** | 伪装成 H2 流量 | ⭐⭐⭐ |

## 选择建议

### 决策树

```
┌─────────────────────────────────────────────────────────────┐
│                      方案选择决策                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  开始                                                       │
│   │                                                         │
│   ├─ 是否有域名？                                           │
│   │   ├─ YES → VLESS+TLS 或 Trojan                         │
│   │   │         ↳ 搭配 CDN 效果更好                         │
│   │   │                                                     │
│   │   └─ NO ↓                                               │
│   │                                                         │
│   ├─ 网络环境是否严格？                                     │
│   │   ├─ YES → VLESS+REALITY                               │
│   │   │         (抗封锁强)                                  │
│   │   │                                                     │
│   │   └─ NO ↓                                               │
│   │                                                         │
│   ├─ 追求速度？                                             │
│   │   ├─ YES → Hysteria2                                   │
│   │   │         (UDP 可能受限)                              │
│   │   │                                                     │
│   │   └─ NO → VLESS+REALITY                                │
│   │                                                         │
└─────────────────────────────────────────────────────────────┘
```

### 场景推荐

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| **个人学习/测试** | VLESS + REALITY | 无需域名，上手快 |
| **长期使用** | VLESS + TLS + CDN | 稳定，抗封 |
| **追求速度** | Hysteria2 | 极速传输 |
| **企业使用** | VLESS + REALITY | 高隐蔽性 |
| **弱网环境** | Hysteria2 | QUIC 优化 |
| **敏感时期** | VLESS + REALITY | 抗封锁强 |
