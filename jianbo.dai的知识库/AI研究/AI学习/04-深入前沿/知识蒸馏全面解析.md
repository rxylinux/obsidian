## 一、什么是知识蒸馏

### 1.1 核心定义

**知识蒸馏** 是一种模型压缩技术，通过让一个小模型（学生）去模仿一个大模型（教师）的行为，将大模型的知识"蒸馏"到小模型中。

```
传统训练：
数据 → 小模型 → 预测
     ↑ 只用硬标签

知识蒸馏：
数据 → 大模型(教师) → 软标签
     ↓              ↓
小模型(学生) → 预测
     ↑ 同时学习硬标签+软标签
```

### 1.2 本质理解

> **知识蒸馏 = 让学生学会老师的"认知方式"**
>
> 不只是"老师得出了什么结论"
>
> 而是"老师是如何得出这个结论的"（思考过程）

就像下棋：
- **传统训练**：只看棋谱（答案）
- **蒸馏训练**：大师讲解每一步为什么这么走（思考过程）

## 二、为什么需要蒸馏

| 场景 | 问题 | 蒸馏的价值 |
|:-----|:-----|:----------|
| **边缘部署** | 手机/物联网算力有限 | 7B → 1B，保持 90%+ 性能 |
| **推理加速** | 大模型延迟高 | 小模型快 10-50x |
| **成本优化** | API 调用昂贵 | 自托管小模型 |
| **多模型集成** | ensemble 太大 | 单个小模型替代 ensemble |

## 三、核心原理

### 3.1 硬标签 vs 软标签

```python
# 硬标签
Hard Label: [0, 0, 1, 0]  # 只有类别2是1，其他都是0
# 问题：信息量太少

# 软标签
Soft Label: [0.01, 0.05, 0.90, 0.04]  # 类别2概率高，但也表达了其他类别的关系
# 优势：包含"暗知识"（Dark Knowledge）
```

**软标签包含的信息：**
- "这主要是猫" (90%)
- "但也有点像狗" (5%)
- "不太像鸟" (1%)

### 3.2 温度参数

温度控制"解释的详细程度"：

```python
def softmax_with_temperature(logits, temperature=1.0):
    """
    温度越高，分布越平滑（软）
    温度越低，分布越尖锐（硬）
    """
    return F.softmax(logits / temperature, dim=-1)

# 示例：logits = [1.0, 2.0, 5.0, 3.0]

T=1（几乎不蒸馏）:   [0.02, 0.05, 0.78, 0.13]  # 只学到"这主要是A"
T=5（中等蒸馏）:     [0.18, 0.20, 0.29, 0.23]  # 学到"A更像B而非D"
T=20（重度蒸馏）:    [0.24, 0.25, 0.26, 0.23]  # 学到"所有类别差不多"
```

**温度选择建议：**
- CV 任务：T = 3-5
- LLM 任务：T = 1-2
- 温度不是越高越好！

### 3.3 损失函数

```python
def distillation_loss(student_logits, teacher_logits, labels, T=3.0, alpha=0.7):
    """
    总损失 = α × 软标签损失 + (1-α) × 硬标签损失
         ↓              ↓
      听老师讲       自己做题对答案
    """
    # 软标签损失（KD Loss）
    soft_loss = F.kl_div(
        F.log_softmax(student_logits / T, dim=1),
        F.softmax(teacher_logits / T, dim=1),
        reduction='batchmean'
    ) * (T ** 2)  # T² 缩放保持梯度

    # 硬标签损失（Cross Entropy）
    hard_loss = F.cross_entropy(student_logits, labels)

    return alpha * soft_loss + (1 - alpha) * hard_loss
```

**为什么两个损失都要？**
- 只听老师：可能老师错了，学生也跟着错
- 只对答案：学不到深层知识
- 两者结合：既学思维，又保准确

## 四、三种蒸馏层次

### 4.1 Response-based（只学输出）

```
老师："答案是C"
     ↓
学生："记住答案是C"

类似：抄答案
```

### 4.2 Feature-based（学中间特征）

```
老师："我看到耳朵是尖的、胡须很长、眼睛是绿色..."
     ↓
学生："我也关注这些特征"

类似：学习方法论
```

### 4.3 Relation-based（学关系）

```
老师："这张猫和那张狗的共同点是...，差异是..."
     ↓
学生："理解了猫和狗的关系"

类似：理解本质
```

## 五、模型蒸馏 vs 数据蒸馏

### 5.1 核心区别

```
模型蒸馏 (Model Distillation)
├─ 需要: 模型权重或中间层访问
├─ 适用于: 开源模型
├─ 学习内容: 中间特征 + 输出分布
├─ 实时性: 实时蒸馏
└─ 效果: ⭐⭐⭐⭐⭐

数据蒸馏 (Data Distillation)
├─ 需要: API访问或数据集
├─ 适用于: 闭源模型
├─ 学习内容: 输出文本（含思考过程）
├─ 实时性: 离线学习
└─ 效果: ⭐⭐⭐⭐
```

### 5.2 对比表

| 维度 | 模型蒸馏 | 数据蒸馏 |
|:-----|:---------|:---------|
| **适用模型** | 开源（有权重） | 开源 + 闭源（API） |
| **需要什么** | 模型权重 | API 或 数据集 |
| **能学到什么** | 中间特征 + 输出 | 只有输出 |
| **训练成本** | 低（计算） | 高（API费用） |
| **数据质量** | 用原始数据 | 需精心设计 |
| **工业界** | 内部优化 | 学习GPT-4能力 |

### 5.3 决策树

```
你想蒸馏什么模型？

├─ 开源模型（有权重）
│   ├─ LLaMA, Qwen, Mistral, BERT...
│   └─ ✅ 用模型蒸馏（效果最好）
│
└─ 闭源模型（只有API）
    ├─ GPT-4, Claude, Gemini...
    └─ ⚠️ 用数据蒸馏（生成训练集）
```

## 六、获取中间层特征

### 6.1 方法一：修改 forward

```python
class TeacherModel(nn.Module):
    def forward(self, x, return_features=False):
        h1 = self.layer1(x)
        h2 = self.layer2(h1)  # 中间特征
        output = self.layer3(h2)

        if return_features:
            return output, h2
        return output

# 使用
teacher = TeacherModel()
output, features = teacher(x, return_features=True)
```

### 6.2 方法二：使用 Hook

```python
# 获取任意层的输出
def get_layer_output(model, layer_name, input_tensor):
    output = None

    def hook(module, input, output):
        nonlocal output
        output = output.detach()

    # 找到目标层
    target_layer = dict(model.named_modules())[layer_name]

    # 注册hook
    handle = target_layer.register_forward_hook(hook)

    # 前向传播
    _ = model(input_tensor)

    # 移除hook
    handle.remove()

    return output

# 使用
model = MyModel()
x = torch.randn(1, 3, 224, 224)
features = get_layer_output(model, "layer3", x)
```

### 6.3 方法三：HuggingFace

```python
from transformers import AutoModel

teacher = AutoModel.from_pretrained("bert-base-uncased")

outputs = teacher(input_ids, output_hidden_states=True)
all_hidden_states = outputs.hidden_states  # 所有层的特征
#            ↑
#     这就是"思考过程"！
```

## 七、模型蒸馏实现

### 7.1 完整训练类

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class DistillationTrainer:
    def __init__(self, teacher, student, temperature=3.0, alpha=0.7):
        self.teacher = teacher.eval()   # 教师不训练
        self.student = student.train()  # 学生训练
        self.temperature = temperature
        self.alpha = alpha

    def train_step(self, batch, optimizer):
        inputs, labels = batch

        # 教师前向传播（无需梯度）
        with torch.no_grad():
            teacher_output = self.teacher(inputs)

        # 学生前向传播
        student_output = self.student(inputs)

        # 计算蒸馏损失
        loss = self.distillation_loss(
            student_output, teacher_output, labels
        )

        # 反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        return loss.item()

    def distillation_loss(self, student_logits, teacher_logits, labels):
        # 软标签损失
        soft_loss = nn.KLDivLoss(reduction='batchmean')(
            nn.LogSoftmax(dim=1)(student_logits / self.temperature),
            nn.Softmax(dim=1)(teacher_logits / self.temperature)
        ) * (self.temperature ** 2)

        # 硬标签损失
        hard_loss = nn.CrossEntropyLoss()(student_logits, labels)

        return self.alpha * soft_loss + (1 - self.alpha) * hard_loss
```

### 7.2 Feature-based 蒸馏

```python
class FeatureDistillationTrainer:
    def __init__(self, teacher, student):
        self.teacher = teacher.eval()
        self.student = student.train()

    def train_step(self, batch, optimizer):
        inputs, labels = batch

        # 获取教师特征
        with torch.no_grad():
            _, teacher_features = self.teacher(inputs, return_features=True)

        # 获取学生特征
        _, student_features = self.student(inputs, return_features=True)

        # 特征对齐损失
        feature_loss = F.mse_loss(student_features, teacher_features)

        # 输出损失
        student_output = self.student(inputs)
        output_loss = F.cross_entropy(student_output, labels)

        # 总损失
        loss = output_loss + 0.5 * feature_loss

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        return loss.item()
```

## 八、数据蒸馏实现

### 8.1 API 数据生成

```python
import openai
import json

class APIBasedDistillation:
    """通过API蒸馏闭源模型"""

    def __init__(self, api_key, model="gpt-4"):
        self.client = openai.OpenAI(api_key=api_key)
        self.model = model

    def generate_dataset(self, prompts, output_file="distillation_data.jsonl"):
        """
        用API生成蒸馏数据
        """
        dataset = []

        for prompt in prompts:
            # 让老师展示思考过程
            response = self.client.chat.completions.create(
                model=self.model,
                messages=[{
                    "role": "user",
                    "content": f"""
请一步步思考并回答这个问题。展示你的思考过程：

{prompt}

请按以下格式：
思考步骤1：...
思考步骤2：...
...
最终答案：...
"""
                }],
                temperature=0.0
            )

            teacher_output = response.choices[0].message.content

            dataset.append({
                "prompt": prompt,
                "teacher_output": teacher_output
            })

            # 保存
            with open(output_file, 'a') as f:
                f.write(json.dumps(dataset[-1]) + '\n')

        return dataset
```

### 8.2 训练学生模型

```python
def train_student_on_dataset(student_model, data_file, epochs=3):
    """
    在生成的数据集上训练学生
    """
    # 加载数据
    dataset = []
    with open(data_file) as f:
        for line in f:
            dataset.append(json.loads(line))

    # 训练
    for epoch in range(epochs):
        for item in dataset:
            # 学生生成
            student_output = student_model.generate(item["prompt"])

            # 和老师输出对比
            loss = compute_loss(
                student_output,
                item["teacher_output"]  # 包含思考过程
            )

            # 反向传播
            loss.backward()

    return student_model
```

## 九、方案选择建议

### 9.1 决策流程

```
你有模型权重吗？

├─ YES → 模型蒸馏
│        │
│        ├─ 简单快速 → Response-based
│        ├─ 追求性能 → Feature-based
│        └─ 极致效果 → Relation-based
│
└─ NO → 有API访问吗？
         ├─ YES → 数据蒸馏
         │        ├─ 设计好的prompt
         │        ├─ 生成高质量数据
         │        └─ 学生学习数据集
         │
         └─ NO → 无法蒸馏
                  └─ 除非找到开源数据集
```

### 9.2 最佳实践

| 方面 | 建议 |
|:-----|:-----|
| **温度 T** | 3-5（CV），1-2（LLM） |
| **α 权重** | 0.5-0.7（软标签优先） |
| **学生容量** | 至少教师的 50% 参数量 |
| **数据质量** | 高质量 > 大数量 |
| **训练阶段** | 先教师再一起训练 |

## 十一、关键要点

```
✅ 开源模型 → 模型蒸馏（学中间层）
✅ 闭源模型 → 数据蒸馏（学输出）
✅ 本质 = 学老师的"思考过程"

✅ 软标签比硬标签包含更多信息
✅ 温度参数控制"解释详细程度"
✅ α 权重平衡软硬标签学习

❌ 蒸馏不是让输出完全相同，而是分布相似
❌ 温度不是越高越好
❌ 闭源模型无法直接获取中间层
```
