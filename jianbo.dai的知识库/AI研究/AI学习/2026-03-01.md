# 神经网络核心概念辨析

> [!info] 背景
> 本笔记记录了对神经网络、架构、模型等核心概念的理解辨析，以及 Transformer 与 RNN 的关系对比。

---

## 一、神经网络、架构、模型的关系

### 1.1 核心概念

```
层次关系：
神经网络（大类技术）
  └─ 架构（具体设计）
      └─ 模型（训练好的实例）
```

### 1.2 用建筑类比

| 概念 | 类比 | 说明 |
|:-----|:-----|:-----|
| **神经网络** | 建筑技术 | 用砖块搭建房子的技术大类 |
| **架构** | 设计图纸 | 具体的房子设计方案 |
| **模型** | 建好的房子 | 按图纸建好的实际房子 |

### 1.3 详细说明

#### 神经网络（大类）

**定义**：一种受人脑神经元启发的计算方法

```
神经网络（大类）
├─ 前馈神经网络（MLP）
├─ 卷积神经网络（CNN）
├─ 循环神经网络（RNN）
├─ Transformer神经网络
└─ 图神经网络（GNN）
```

**特点**：
- 是**技术类别**，不是具体实现
- 定义了"用神经元连接来计算"的基本思想

#### 架构（具体设计）

**定义**：神经网络的具体结构设计

```
架构 = 设计方案
├─ 有多少层？
├─ 每层什么类型？
├─ 层之间怎么连接？
└─ 用什么激活函数？
```

**示例**：
- ResNet-50：50层的残差网络设计
- LSTM：2层、隐藏维度128的配置
- Transformer（原始版）：Encoder-Decoder结构

#### 模型（训练好的实例）

**定义**：用特定数据训练后的架构

```
模型 = 架构 + 训练好的参数
├─ 架构：设计方案
├─ 参数：具体的权重数值
└─ 能力：可以完成特定任务
```

**示例**：
- `torchvision.models.resnet50(pretrained=True)`：在ImageNet上训练好的ResNet-50
- `Qwen-72B`：用大量数据训练好的72B参数模型
- `Gemini Pro`：Google训练好的多模态模型

### 1.4 完整示例链条

#### 示例1：图像识别

```
1. 神经网络（大类）
   └─ "我用卷积神经网络（CNN）来处理图像"

2. 架构（设计）
   └─ "我用 ResNet-50 架构"
       ├─ 50层卷积层
       ├─ 残差连接
       └─ 特定结构设计

3. 模型（实例）
   └─ "ImageNet预训练的 ResNet-50"
       ├─ 架构：ResNet-50
       ├─ 参数：2500万个训练好的权重
       └─ 能力：识别1000类物体
```

#### 示例2：大语言模型

```
1. 神经网络（大类）
   └─ "我用 Transformer 神经网络"

2. 架构（设计）
   └─ "Decoder-only Transformer"
       ├─ 注意力机制
       ├─ 位置编码
       └─ 前馈网络

3. 模型（实例）
   └─ "Qwen-72B"
       ├─ 架构：Decoder-only Transformer
       ├─ 参数：720亿个训练好的权重
       └─ 能力：对话、写作、编程
```

### 1.5 对比总结表

| 维度 | 神经网络 | 架构 | 模型 |
|:-----|:---------|:-----|:-----|
| **本质** | 技术类别 | 设计方案 | 训练好的实例 |
| **类比** | 建筑技术 | 设计图纸 | 建好的房子 |
| **状态** | 抽象概念 | 具体结构 | 可用的产物 |
| **可数性** | 不可数 | 可数（不同架构）| 可数（不同实例）|
| **参数** | 不涉及 | 结构参数 | 权重参数 |
| **例子** | CNN、RNN | ResNet、LSTM | resnet50(pretrained)、Qwen |

## 二、Transformer 的双重身份

### 2.1 为什么特殊？

```
Transformer 有两个身份：

身份1：神经网络类型（像 CNN、RNN 一样的分类）
├─ CNN = 卷积神经网络
├─ RNN = 循环神经网络
└─ Transformer = Transformer神经网络

身份2：具体架构（2017年论文提出的设计）
└─ 论文《Attention Is All You Need》提出的设计就叫 "Transformer"
```

### 2.2 与 CNN、RNN 的对比

```
CNN 的情况：
├─ CNN = 神经网络类型（卷积神经网络）
├─ 没有一个架构叫 "CNN"
└─ 有 ResNet、VGG、AlexNet 等具体架构

RNN 的情况：
├─ RNN = 神经网络类型（循环神经网络）
├─ 没有一个架构叫 "RNN"（只有 Vanilla RNN，但很少用）
└─ 有 LSTM、GRU 等具体架构

Transformer 的情况：
├─ Transformer = 神经网络类型
├─ 有一个架构就叫 "Transformer"（原始版，2017年）
└─ 还有 BERT、GPT、T5 等衍生架构
```

### 2.3 Transformer 架构家族

```
Transformer 架构家族：
├─ Encoder-only（仅编码器）
│   └─ BERT、RoBERTa
│   └─ 特点：双向理解，擅长理解任务
│
├─ Decoder-only（仅解码器）
│   └─ GPT系列、Qwen、LLaMA、Claude、Gemini
│   └─ 特点：单向自回归，擅长生成任务
│
└─ Encoder-Decoder（编码器-解码器）
    └─ T5、BART
    └─ 特点：编码理解+解码生成，擅长翻译
```

### 2.4 现代大模型架构

| 模型 | 基础架构 | 具体变体 |
|:-----|:---------|:---------|
| **GPT-4** | Transformer | Decoder-only |
| **Claude** | Transformer | Decoder-only |
| **Qwen** | Transformer | Decoder-only + GQA + RoPE |
| **LLaMA** | Transformer | Decoder-only + GQA + RoPE |
| **Gemini** | Transformer | Decoder-only + 多模态 |
| **BERT** | Transformer | Encoder-only |

**共同点**：都基于 **Transformer** 架构，现代大模型基本都是 **Decoder-only Transformer**

## 三、Transformer 与 RNN 的对比

### 3.1 本质区别

```
问题：Transformer 算是 RNN 一种吗？

答案：❌ 不算

正确关系：
├─ RNN = 循环神经网络（一类）
├─ Transformer = Transformer神经网络（另一类）
└─ 它们是平行的两类神经网络，不是包含关系

类比：
├─ RNN = "中式建筑风格"
├─ Transformer = "现代建筑风格"
└─ 两者都是建筑，但风格完全不同
```

### 3.2 核心差异

| 维度 | RNN (LSTM/GRU) | Transformer |
|:-----|:---------------|:-------------|
| **计算方式** | 串行（一个接一个） | 并行（全部同时） |
| **信息传递** | 隐状态逐步传递 | 自注意力全局连接 |
| **时序建模** | 显式的时序递归 | 位置编码隐式时序 |
| **长距离依赖** | 渐弱（梯度消失） | 保持不变（直达） |
| **训练速度** | 慢（无法并行） | 快（高度并行） |
| **推理速度** | 快（O(seq_len)） | 慢（O(seq_len²)） |

### 3.3 信息流动对比

```
场景：理解长句中的指代关系

RNN 的路径：
词₁ ──► ... ──► 词ₙ
  ↑                   ↑
  └────── 长路径 ──────┘
  （信息经过多个时间步，可能衰减）

Transformer 的路径：
词₁ ══════════════ 词ₙ
  ↑                    ↑
  └─── 直接连接 ───────┘
  （一步到位，无衰减）
```

### 3.4 发展时间线

```
2017前：RNN 时代
  RNN → LSTM → GRU
  （处理序列的主流方法）

2017：Transformer 横空出世
  《Attention Is All You Need》
  └─ "只需要注意力，不需要RNN"

2018后：Transformer 时代
  BERT、GPT、T5 等全部基于 Transformer
  RNN 在NLP逐渐被取代
```

## 四、股票走势预测的模型选择

### 4.1 问题性质

```
股票走势分析 = 时间序列预测问题

核心特点：
├─ 时间序列数据
├─ 非线性、非平稳
├─ 噪声大、波动性高
└─ 受多种因素影响
```

### 4.2 模型适配度对比

| 模型 | 适配度 | 优势 | 劣势 | 适用场景 |
|:-----|:-------|:-----|:-----|:---------|
| **LSTM/GRU** | ⭐⭐⭐⭐⭐ | 专为时序设计 | 训练较慢 | **首选**，大多数场景 |
| **Transformer** | ⭐⭐⭐⭐⭐ | 并行、长距离依赖 | 数据需求大 | **首选**，有足够数据 |
| **TCN** | ⭐⭐⭐⭐ | 并行、感受野灵活 | 超长序列有限 | 中短期预测 |
| **CNN** | ⭐⭐⭐ | 快速、局部模式 | 时序建模弱 | 提取K线形态 |
| **GNN** | ⭐⭐⭐ | 考虑板块关联 | 构建复杂 | **进阶**，多股分析 |

### 4.3 三大主流方案

#### 方案1：LSTM/GRU（最经典）

```python
class StockLSTM(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_layers, output_dim):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        # x: [batch, seq_len, features]
        lstm_out, _ = self.lstm(x)
        return self.fc(lstm_out[:, -1, :])
```

**特点**：
- ✅ 成熟稳定，工业界验证多
- ✅ 处理长短期依赖
- ❌ 训练慢（串行计算）

#### 方案2：Transformer（最前沿）

```python
class StockTransformer(nn.Module):
    def __init__(self, input_dim, d_model=64, nhead=4, num_layers=3):
        super().__init__()
        self.embedding = nn.Linear(input_dim, d_model)
        self.pos_encoding = PositionalEncoding(d_model)

        encoder_layer = nn.TransformerEncoderLayer(d_model, nhead)
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers)
        self.fc = nn.Linear(d_model, 1)

    def forward(self, x):
        x = self.embedding(x)
        x = self.pos_encoding(x)
        x = self.transformer(x)
        return self.fc(x[:, -1, :])
```

**特点**：
- ✅ 并行训练，速度快
- ✅ 捕捉长距离依赖
- ❌ 需要较多数据

#### 方案3：混合模型

```python
class CNNLSTM(nn.Module):
    """结合CNN的形态识别和LSTM的时序建模"""
    def __init__(self, input_dim, hidden_dim):
        super().__init__()
        # CNN提取局部形态
        self.conv1 = nn.Conv1d(input_dim, 32, kernel_size=3)
        self.conv2 = nn.Conv1d(32, 64, kernel_size=3)
        # LSTM时序建模
        self.lstm = nn.LSTM(64, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, 1)

    def forward(self, x):
        # x: [batch, seq_len, features]
        x = x.transpose(1, 2)  # [batch, features, seq_len]
        x = self.conv1(x).relu()
        x = self.conv2(x).relu()
        x = x.transpose(1, 2)  # [batch, seq_len, 64]
        x, _ = self.lstm(x)
        return self.fc(x[:, -1, :])
```

### 4.4 选择指南

```
┌─────────────────────────────────────────────────────┐
│              股票预测模型选择指南                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 数据量如何？                                     │
│     ├─ 少（<1万条）     → LSTM/GRU                 │
│     ├─ 中等（1-10万）   → LSTM、TCN                │
│     └─ 多（>10万）      → Transformer              │
│                                                     │
│  2. 预测时序长度？                                   │
│     ├─ 短期（1-5天）    → LSTM、TCN                │
│     ├─ 中期（5-20天）   → LSTM、Transformer        │
│     └─ 长期（>20天）    → Transformer              │
│                                                     │
│  3. 是否需要考虑股票关联？                           │
│     ├─ 是               → GNN + LSTM/Transformer   │
│     └─ 否               → 单股票模型                │
│                                                     │
│  4. 计算资源？                                       │
│     ├─ 有限             → LSTM、TCN                │
│     └─ 充足             → Transformer              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 4.5 重要提醒

**股票预测的核心难点**：
1. **非平稳性** - 市场规律不断变化
2. **噪声极大** - 随机性很强
3. **外部因素** - 政策、情绪等难以量化
4. **过拟合风险** - 历史表现≠未来

**实用建议**：
- 不要期待高准确率（55-60%就不错了）
- 结合基本面分析
- 严格的风险管理
- 模型 ensemble 提高稳定性

## 五、关键要点总结

### 5.1 概念层次

```
神经网络家族：

1. CNN 家族
   - 特点：卷积操作
   - 架构：ResNet、VGG、AlexNet

2. RNN 家族
   - 特点：循环递归
   - 架构：LSTM、GRU

3. Transformer 家族
   - 特点：注意力机制
   - 架构：Transformer（原始）、BERT、GPT、T5

4. GNN 家族
   - 特点：图结构处理
   - 架构：GCN、GAT、GraphSAGE
```

### 5.2 Transformer 的特殊性

```
Transformer 特殊之处：

既是类型又是架构：
├─ Transformer神经网络（类型，像CNN、RNN）
└─ Transformer架构（具体设计，2017年论文）

原因：
└─ 2017年论文直接用 "Transformer" 命名了这个架构
```

### 5.3 现代大模型格局

```
现代大语言模型（LLM）：

基础架构：Decoder-only Transformer
├─ GPT-4（OpenAI）
├─ Claude（Anthropic）
├─ Qwen（阿里）
├─ LLaMA（Meta）
└─ Gemini（Google）

共同特点：
├─ 都是具体模型（训练好的）
├─ 都用 Decoder-only Transformer
├─ 各自优化：GQA、RoPE、SwiGLU 等
└─ 越大越好（Scaling Law）
```
